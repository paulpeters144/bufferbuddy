1. INTRODUCTION                                    *bufferbuddy-introduction*

BufferBuddy is an AI coding assistant for Neovim powered by Google Gemini
and Claude. It provides an interactive chat interface and inline
code editing using natural language instructions, with access to code
search tools (ripgrep, ast-grep) for context-aware assistance.

Features:
  - Interactive AI chat window with code search capabilities
  - Inline code editing via natural language instructions
  - Code explanation (single line or selection)
  - Automatic selection expansion to enclosing function/class
  - Tool-supported LLM: rg, ast-grep, find_definitions

==============================================================================
2. REQUIREMENTS                                    *bufferbuddy-requirements*

  - Neovim >= 0.10
  - plenary.nvim (https://github.com/nvim-lua/plenary.nvim)
  - ripgrep (rg) for code search tool
  - ast-grep (sg) for structural code search (optional but recommended)
  - A Google Gemini API key or Anthropic API key (for Claude)

==============================================================================
3. INSTALLATION                                    *bufferbuddy-installation*

Using lazy.nvim: >
  {
    "username/bufferbuddy",
    dependencies = { "nvim-lua/plenary.nvim" },
    opts = {
      api_key = "your-api-key",
    },
  }
<
For Claude: >
  {
    "username/bufferbuddy",
    dependencies = { "nvim-lua/plenary.nvim" },
    opts = {
      provider = "claude",
      api_key = "your-anthropic-api-key",
    },
  }
<
==============================================================================
4. SETUP                                                       *bufferbuddy-setup*

Call the setup function in your Neovim config: >
  -- Gemini (default)
  require("bufferbuddy").setup({
    api_key = "GEMINI_API_KEY",        -- or set GEMINI_API_KEY env var
    provider = "gemini",               -- default
    model = "gemini-3.1-flash-lite",   -- model name
    max_tool_rounds = 15,              -- max LLM tool call iterations
  })

  -- Claude
  require("bufferbuddy").setup({
    api_key = "ANTHROPIC_API_KEY",     -- or set ANTHROPIC_API_KEY env var
    provider = "claude",
    model = "claude-3-5-haiku-20241022", -- model name (fastest/cheapest)
    max_tool_rounds = 15,              -- max LLM tool call iterations
    max_tokens = 8192,                 -- max response tokens
  })
<

Set the `GEMINI_API_KEY` or `ANTHROPIC_API_KEY` environment variable
to skip the `api_key` in setup().

==============================================================================
5. COMMANDS                                                *bufferbuddy-commands*

  :BufferBuddyChat    Open the interactive AI chat window

==============================================================================
6. KEYMAPS                                                  *bufferbuddy-keymaps*

Default keymaps (set in |plugin/bufferbuddy.lua|):

  <leader>bbq  Normal: Open chat window
  <leader>bbq  Visual: Open chat with selected lines
  <leader>bbx  Normal: Explain the current line
  <leader>bbx  Visual: Explain the selected lines
  <leader>bbe  Normal: Edit the current line via LLM
  <leader>bbe  Visual: Edit the selected lines via LLM

==============================================================================
7. CONFIGURATION                                    *bufferbuddy-configuration*

The entire configuration is passed to |require("bufferbuddy").setup()|.

  Config key       Default                     Description ~
  api_key           nil                         API key (or GEMINI_API_KEY / ANTHROPIC_API_KEY env var)
  provider          "gemini"                    LLM provider ("gemini" or "claude")
  model             auto                        Model (auto-selected per provider)
  max_tool_rounds   15                          Max function-calling rounds
  max_tokens        nil                         Max response tokens (defaults to 8192 for Claude)

==============================================================================
8. HEALTH                                                    *bufferbuddy-health*

Run |:checkhealth bufferbuddy| to verify your setup. The health check
reports on:
  - Module loading
  - Model and API key configuration
  - ripgrep installation
  - ast-grep installation
